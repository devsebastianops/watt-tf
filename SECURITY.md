# Security Policy for Watt TF

We take the security of tool seriously. If you find a security vulnerability in Watt TF, please let us know immediately so we can fix it before it is publicly disclosed.

Thank you for helping keep the developer and platform community safe! ❤️

## Supported Versions

Since Watt TF is a fast-moving project, we actively support and patch only the latest major/minor release.

## Reporting a Vulnerability

> [!IMPORTANT]
> **Do not open GitHub Issues for security vulnerabilities.** Publicly disclosing a bug before a patch is ready puts other teams and their infrastructure at risk.

If you discover a security-related issue (such as an execution escape in Google CEL, potential credential leakage in environment variables, or a dependency vulnerability), please report it privately:

1.  Send an email to **[security@devsebastianops.com](mailto:security@devsebastianops.com)**.
2.  Include as much detail as possible, including:
    -   The version of Watt TF you used.
    -   A brief description of the vulnerability.
    -   Steps to reproduce the issue (a minimal `input.yaml` and `blueprint.yaml` if applicable).
3.  We will acknowledge your report within **48 hours** and keep you updated on our progress toward a fix.

## Our Process

- **Triage**: We verify the vulnerability and assess the risk level.
- **Fix**: We work on a patch to fix the issue.
- **Release**: We release a new version of Watt TF with the fix and update our documentation.
- **Credit**: Unless you prefer to remain anonymous, we will gladly credit you in our release notes and changelog for your help in keeping Watt TF secure!
